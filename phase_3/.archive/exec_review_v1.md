### Issues Found
- [HIGH] Mixed-status overlap can wrongly downgrade a still-paid organization — `src/modules/entitlements/service.ts:74-116` — `recomputeForOrganization()` picks the first "active paid" subscription by `currentPeriodEnd` sort order and maps that single row to the entitlement. I reproduced a case with one `ACTIVE` subscription and one later-ending `PAYMENT_ISSUE` subscription; both `/v1/billing/subscription` and `/v1/entitlements/current` returned `payment_issue`, even though the locked policy says any valid active paid source should keep access paid and only set `billing_overlap=true`. Fix: replace sort-order primary selection with explicit entitlement precedence across overlapping paid statuses, then compute `billing_overlap` separately. Verify with regression tests for `ACTIVE + PAYMENT_ISSUE`, `ACTIVE + GRACE`, and `ACTIVE + TRIALING`.
- [MEDIUM] `/v1/entitlements/current` can return `free` while an active Stripe subscription already exists — `src/modules/entitlements/service.ts:37-52`; compare `src/modules/billing/service.ts:103-111` — the entitlements read path returns a hardcoded free default when the entitlement row is missing, but `/v1/billing/subscription` recomputes from the subscription table when necessary. I reproduced this by seeding an `ACTIVE` Stripe subscription without an entitlement row: `/v1/entitlements/current` returned `free`, while `/v1/billing/subscription` returned `pro_active`. Fix: make `getCurrentEntitlement()` recompute when the row is missing, or share the same recomputing read-model path between both endpoints. Verify with a regression test that hits `/v1/entitlements/current` first in this state.
- [LOW] Phase 3 billing coverage does not exercise the failing edge cases above — `test/integration/billing.stripe.test.ts:132-204` — the overlap test only seeds two `ACTIVE` subscriptions, and it calls `/v1/billing/subscription` before `/v1/entitlements/current`, which hides the missing-row divergence. Fix: add explicit regression coverage for mixed-status overlaps and for entitlements-first reads when only subscription rows exist.
